# Real-Time Face Detection — Video Streaming System

> **Backend Internship Take-Home Assignment**
> FastAPI · MediaPipe · Pillow · PostgreSQL · React · Docker

A fully containerised system that accepts a live webcam feed over WebSocket, detects faces using **MediaPipe** (zero OpenCV), draws bounding boxes with **Pillow**, stores Region-of-Interest (ROI) data in **PostgreSQL**, and streams the annotated feed back to a browser viewer in real time.

---

## Architecture

```
Browser (React/Vite)
│
├── /stream/upload  ─── WebSocket ──▶  FastAPI  ──▶  MediaPipe + Pillow
│                                          │               │
├── /stream/view   ◀── WebSocket ─────────┘               ▼
│                                                     PostgreSQL
└── /api/v1/roi   ◀── REST (GET) ─────────────────────────┘
```

Three Docker services, two Docker networks:

| Service | Image | Network(s) |
|---|---|---|
| `db` | postgres:15-alpine | `internal` only — **not** exposed to host |
| `backend` | python:3.11-slim | `internal` + `external` |
| `frontend` | nginx:1.27-alpine | `external` only |

## Quickstart (Docker — recommended)

```bash
# 1. Clone
git clone https://github.com/CrazyPrash007/realtime-face-detection.git
cd realtime-face-detection

# 2. Configure secrets (never committed)
cp .env.example .env
# Edit .env if you want a custom POSTGRES_PASSWORD

# 3. Build and run all services
docker-compose up --build

# 4. Open the app
open http://localhost
```

Allow camera access when prompted. The left panel shows your raw webcam feed; the right panel shows the processed stream with green bounding boxes around detected faces.

## API Reference

### WebSocket — Upload Stream

```
WS /stream/upload
```

**Protocol:**
1. Connect (no query params).
2. Server sends `{"session_id": "<uuid>"}` — the backend owns this ID.
3. Client sends raw JPEG bytes at the desired frame rate.
4. Server processes each frame and broadcasts it to all viewers in the same session.

### WebSocket — View Stream

```
WS /stream/view?session_id=<uuid>
```

Connects a viewer to receive processed JPEG frames for the given session. The `session_id` must be the one received from the upload socket's first message.

### REST — List ROI Events

```
GET /api/v1/roi
```

| Parameter | Type | Default | Constraints | Description |
|---|---|---|---|---|
| `session_id` | string | — | optional | Filter to a specific session |
| `limit` | int | 50 | 1–500 | Records per page |
| `offset` | int | 0 | ≥ 0 | Pagination offset |

**Response:**
```json
{
  "total": 142,
  "limit": 50,
  "offset": 0,
  "items": [
    {
      "id": "uuid",
      "session_id": "uuid",
      "timestamp": "2025-05-03T10:00:00Z",
      "x": 120, "y": 80, "width": 200, "height": 240
    }
  ]
}
```

### Health Probe

```
GET /health  →  {"status": "ok"}
```

Used by Docker healthcheck. Returns 200 when the backend is ready.

## Security

| Control | Implementation |
|---|---|
| **Secrets** | Managed via `.env`, validated by `pydantic_settings`. Real `.env` is `.gitignore`d. |
| **CORS** | Strict allow-list from `ALLOWED_ORIGINS` env var. No `*` wildcard. |
| **Session isolation** | `session_id` is generated by the backend, not the client — prevents stream hijacking. |
| **Frame size cap** | Frames > `MAX_FRAME_BYTES` (default 1 MB) are dropped immediately. |
| **IP rate limiting** | Max `MAX_WS_CONNECTIONS` (default 10) active connections per IP. |
| **DB isolation** | PostgreSQL runs on an `internal` Docker network — its port is never exposed to the host. |
| **Least privilege** | A dedicated `faceuser` has only `CONNECT`, `SELECT`, and `INSERT` — no DDL. |
| **Non-root containers** | Both backend (`appuser`) and frontend (nginx default) run as non-root. |
| **Event loop** | MediaPipe runs in a thread-pool executor — CPU work never blocks async I/O. |

## Running Tests

```bash
cd backend
pip install -r requirements.txt
pytest -v
```

No running PostgreSQL is required — tests use an in-memory SQLite engine.

Test files:

| File | What it covers |
|---|---|
| `tests/test_roi_endpoint.py` | REST endpoint schema, pagination bounds, session filter |
| `tests/test_face_service.py` | No-face passthrough, detection path, corrupt input handling |
| `tests/test_connection_manager.py` | Session pairing, IP rate limit, broadcast, stale-connection pruning |
